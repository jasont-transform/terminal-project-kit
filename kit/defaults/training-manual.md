# Cross-project training manual

Lessons, insights, techniques, and patterns that apply across more than one project. Per-project manuals live inside each project's `design-journal/training-manual.md`.

## How this file is maintained

An entry lives here only if it is useful across at least two projects. Project-specific craft stays in the project's own manual. When a cross-cutting lesson emerges — typically when we notice "this applies elsewhere too" — lift it here and leave a pointer in the project manual.

Entries follow the same four shapes as per-project manuals:

- **Lesson** — a principle learned the hard way.
- **Insight** — a conceptual reframe.
- **Technique** — a how-to or decision heuristic.
- **Pattern** — a reusable structural template.
