# Glossary — Demo project

Short, unambiguous definitions of project-specific vocabulary. Add a new `## Term` section whenever a word enters the project vocabulary that a first-time reader could misinterpret.

## Demo project

The pre-populated example project shipped by the Claude Project Kit. It exists so a new user can see what a healthy `design-journal/` looks like before touching their own. You are reading its glossary right now.

## Logbook

The chronological prose record of a project, one Markdown file per day in `design-journal/logbook/`. Written during the session and supplemented by the Stop hook at the end of each session. Private by default (gitignored). Contrast with *training manual*, which is timeless and public.

## Training manual

The audience-facing knowledge base for a project, at `design-journal/training-manual.md`. Organised by topic, not chronologically. A new team member should be able to pick up the craft from this document without reading the logbook.
